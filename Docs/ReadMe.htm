Description

Overview

TPJHotLabel is a simple Delphi component that descends from TLabel. It provides a clickable label that can start the default browser or email client to access a specific URL. Key features of the component are:

  • All properties of TLabel are supported, although the usage and default values of some of the inherited properties have been changed.
  • The URL property is used for storing the URL to be accessed when the label is clicked and the Caption property can either store descriptive text or can reflect the URL, depending on the value of the CaptionIsURL property. The ability to link the Caption and URL makes it easy to display the URL without having to keep two properties synchronised.
  • The URL can be validated to check for supported protocols. This validation is switched on and off using the ValidateURL property. Supported protocols are:
    • http://
    • https://
    • mailto:
    • file:
    • ftp://
  • The label's Font property defaults to a navy blue to indicate a link.
  • The label can be highlighted when the mouse passes over it. Highlighting is used if the HighlightURL property is True, and the font used for highlighting is set using the HighlightFont property.
  • The label displays the "hand point" cursor by default. For versions of Delphi that do not support provide the crHandPoint cursor, a custom cursor is linked in to the program by the component.
  • The component's pop-up hints can be customised as follows, using the HintStyle property:
    • the hint text can come from the Hint property as normal,
        or
    • the hint text can come from the URL property with the Hint property being ignored,
        or
    • the hint text can be modified just before the hint is shown by handling the OnCustomHint event, which is useful for displaying dynamic information in the hint.

A help file is provided that can integrate with the IDE of Delphi 3 to 7.

Backwards compatibility with v1

This v2 release is backwards compatible with v1 of the component. All new properties that are not available in v1 are either switched off by default or emulate the properties available in v1.

Compiler Compatibility

TPJHotLabel has been tested Delphi 4, 6, 7 and 2006. It should work with all Win32 versions of Delphi (i.e. versions 2 to 2006). The component is not suitable for use with Delphi 1.

Installation

TPJHotLabel, its help files and documentation are supplied in a zip file. Before installing you need to extract all the files from the zip file, preserving the directory structure. The following files will be extracted:

  • PJHotLabel.pas – Component source code.
  • PJHotLabel.dcr – Resource file containing a 32 bit component palette glyph.
  • PJHotLabel.rc – Source of resource file that contains cursor required for use with Delphi 2 and 3.
  • PJHotLabel.res – Resource file containing a cursor required for use with Delphi 2 and 3.
  • Hand.cur – Hand point cursor required for use with Delphi 2 and 3.
  • PJHotLabel.hlp – Component help file that integrates with the Delphi 3 to 7 OpenHelp system.
  • PJHotLabel.als – A-link keyword file for integration with the Delphi 6 or 7 OpenHelp system.
  • ReadMe.htm – Component information file.
  • ChangeLog.txt – Component's change log.
  • MPL.txt – Mozilla Public License v1.1.
  • Demo\HotLabelDemo.dpr – Demo program project file.
  • Demo\HotLabelDemo.res – Demo program resource file containing program icon.
  • Demo\FmHotLabelDemo.pas – Demo program's main form source code.
  • Demo\FmHotLabelDemo.dfm – Demo program's main form resource file.
  • Demo\DelphiDabbler.ico – Demo program's icon.

You can now proceed to install the component using the emphasised files above. Installation methods vary between Delphi versions, so follow the instructions that apply to your version below. Note that $(DELPHI) represents the path where the appropriate version of Delphi was installed. Similarly $(BDS) refers to the installation path of a version of the Borland Development System.

  • Installation with Delphi 2005 & 2006
  • Installation with Delphi 4 to 7
  • Installation with Delphi 3
  • Installation with Delphi 2

Installation with Delphi 2005 & 2006

The component must be installed into a Delphi Win32 design package. It is assumed that you will be installing into an existing package. You may decide to install into the "user" package: $(BDS)\Lib\dclusr.dpk.

  1. Decide upon a suitable location into which to install the component. This may be a subdirectory of your Borland Studio Projects or $(BDS)\lib folders.
  2. Copy PJHotLabel.pas and PJHotLabel.dcr to the chosen installation folder.
  3. Start the Development System (selecting the Delphi Win32 personality if appropriate) then:
    1. Open the required package. Select the File | Open menu and choose a suitable package source file.
    2. The package (with .bpl extension) will now be displayed in the Project Manager. If the package has not been installed right click the .bpl entry in the Project Manager and select Install. Dismiss any resulting message dialog.
    3. Now right click the .bpl entry again and select Add. The "Add" dialog box will be displayed. Click the dialog's Browse button then select PJHotLabel.pas from the resulting file open dialog. The full path to the file should now be displayed in the "Add" dialog's Unit file name edit box. Click OK to add the component to the package.
    4. Back in the Project Manager, check that PJHotLabel.pas and PJHotLabel.dcr appear under the package's "Contains" node. If so, right click the .bpl entry again and select Compile. The package will now be recompiled and, if all is well, a message should be displayed indicating that TPJHotLabel has been registered. The component should now appear in the "DelphiDabbler" section of the Tool Palette. (Note this will only appear when a form is selected in the designer).
    5. Close the package project using the File | Close menu item. If a prompt appears offering to save the package project, accept this.
  4. If you wish you can now delete PJHotLabel.pas from the place where you copied it.
  5. The supplied help file does not integrate into the Delphi 2005 or 2006 IDEs. The help file can be used stand-alone, but be aware that it contains links into other Delphi help files that will not work.
  6. The demo program can be loaded into Delphi and compiled and run as required.

Installation with Delphi 4 to 7

The component must be installed into a design package. It is assumed that you will be installing into an existing package. You may wish to use the "user" package named dclusr.dpk or dclusrXX.dpk (where XX is a version number). This package can be found in $(DELPHI)\Lib.

  1. Decide upon a suitable location into which to install the component. This may be a subdirectory of your $(DELPHI)\lib folder.
  2. Copy PJHotLabel.pas and PJHotLabel.dcr to the chosen installation folder.
  3. Start Delphi then:
    1. Choose the Component | Install Component menu option to display the Install Component dialog box. Ensure the Into existing package tab is selected. Enter the full path to PJHotLabel.pas in the Unit file name edit box (use the adjacent Browse button to navigate to the file). Choose the required package from the Package description combo box, or by navigating to the package using Browse. Click OK to confirm your selection.
    2. A confirmation dialog box may displayed, querying whether it is OK to rebuild and reinstall the package. Click Yes to proceed.
    3. The package will now be built and a dialog box should be displayed stating that the TPJHotLabel component has been installed. Click OK to dismiss the dialog box.
    4. The component should now appear in the Component Palette on the "DelphiDabbler" tab.
  4. If you wish you can now delete PJHotLabel.pas from the place where you copied it.
  5. The provided help file can be integrated with the IDE's OpenHelp system. Delphi 6 and 7 also require the use of the supplied .als file. The simplest way to install the help file is to use my Component Help Installer Expert (v2.2.1 or later). If you prefer to install manually, this article explains the process.
  6. The demo program can be loaded into Delphi and compiled and run as required. For Delphi version before v7, unsupported form properties may need to be removed in order to compile the program.

Choose the required package from the Package description combo box, or by navigating to the package using Browse. Click OK to confirm your selection.
    2. A confirmation dialog box may displayed, querying whether it is OK to rebuild and reinstall the package. Click Yes to proceed.

    4. The component should now appear in the Component Palette on the "DelphiDabbler" tab.

Installation with Delphi 2

  1. Decide upon a suitable location into which to install the component. This may be a subdirectory of your $(DELPHI)\lib folder.
  2. Copy PJHotLabel.pas, PJHotLabel.dcr and PJHotLabel.res to the chosen installation folder.
  3. Start Delphi then:
    1. Select the Component | Install menu option to display the Install Components dialog box.
    2. Click the Add button to display the Add Module dialog box.
    3. Click the Browse button and choose PJHotLabel.pas from the resulting standard file open dialog box and click Open.
    4. This returns you to the Install Components dialog box. Click OK here to rebuild the component library.
    5. TPJHotLabel should now appear on the "DelphiDabbler" tab of the component palette.
  4. If you wish you can now delete PJHotLabel.pas from the place where you copied it.
  5. The supplied help file does not integrate into the Delphi 2 IDE. The help file can be used stand-alone, but be aware that it contains links into other Delphi help files that will not work.
  6. The demo program can be loaded into Delphi and compiled and run as required. Unsupported form properties may need to be removed in order to compile the program.
